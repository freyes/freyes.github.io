Open Source Project Development Tools: part 1

Mon 27 January 2014
By Felipe Reyes

This is a series of article to explain the tools I choose to use when I start a new project.

Version Control System

I've used for professional development Subversion (svn), Mercurial (hg) and Git, I've reached a decent proficiency with all of them, and I have to say that Git is the best fit for me, no news you'll say, but thanks to GitHub the popularity of Git sky rocketed and a lot of people is being benefited by the popularity of Github.

I'm not implying that developers using other tools are wrong, but you have to recognize that these use CVS, Perforce or even svn is a nightmare for programmers that already felt the privileges of using a VCS like Git or Mercurial.

Why Git?

There are as many reasons as git users, but these are some of my favorite features on my day to day work.

  • magit, it's a git interface for Emacs, it makes the use of git inside Emacs a pleasure.

  • GitHub, free for open source, easy to use for closed source development, Travis integration (more on this later). wiki and bug tracker.

  • git bisect, have you ever had to deal with a regression and you have a point in time where a feature works, and another one where it doesn't?, well git bisect let you do a binary search easily. I recommend you to read "Fighting regressions with git bisect" by Christian Couder

  • git grep, it does what you expect from it (grep over all the versioned files), if you are coming from svn then you can drop ack-grep.

  • Jenkins Git Plugin, it's way better than the plugin for Mercurial, it has the option to look for changes without having a copy of the repo (see git ls-remote), this let you keep you jenkins slaves turned off and the master takes care of polling for changes, once a new change appears then the slave is turned on and the build is triggered.

On the next part of this series I'll be covering continuous integration tools and the workflows.
